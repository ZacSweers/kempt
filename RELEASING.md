# Releasing

## Regular release

```sh
./release.sh 0.2.0
```

The script handles the full process:

1. Bumps the version in `Cargo.toml`.
2. Promotes the `[Unreleased]` section in `CHANGELOG.md` to a dated version
   heading.
3. Builds to refresh `Cargo.lock`.
4. Commits and pushes to `main`.
5. Tags and pushes the version tag, which triggers the release workflow.
6. Watches the release workflow until it finishes.
7. Prompts to publish to crates.io.

### Prerequisites

- Clean working tree.
- `gh` CLI installed and authenticated.
- `cargo login` already run, if you want to publish to crates.io.

## Upgrading cargo-dist

The release workflow at `.github/workflows/release.yml` is autogenerated by
`dist generate`. Never hand-edit it.

When a new cargo-dist version is available:

1. Bump `cargo-dist-version` in `dist-workspace.toml`.
2. Regenerate:
   ```sh
   dist generate
   ```
3. Verify:
   ```sh
   dist generate --check
   dist plan
   ```
4. Commit `dist-workspace.toml` and `release.yml` together.

## Secrets

These GitHub Actions secrets must be configured on `ZacSweers/kempt`:

| Secret               | Purpose                                                                                                               |
|----------------------|-----------------------------------------------------------------------------------------------------------------------|
| `HOMEBREW_TAP_TOKEN` | PAT with `Contents: read/write` on `ZacSweers/homebrew-tap`. Used by the release workflow to push the formula update. |

For crates.io publishing, the script uses your local `cargo login` session.
No secret needed in CI.
